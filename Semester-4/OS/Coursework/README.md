## Описание работы

Работа посвящена практическому применению механизмов безопасности операционной системы Linux для защиты инфраструктурного компонента Web3 — Ethereum-ноды. Ethereum-нода представляет собой процесс, взаимодействующий с сетью и потенциально имеющий доступ к криптографическим ключам. Компрометация такого процесса создаёт угрозу активам и инфраструктуре.

**Цель:** продемонстрировать применение штатных механизмов ядра Linux для минимизации поверхности атаки и ограничения последствий возможной компрометации.

---

## Порядок выполнения

### 1. Развёртывание тестовой среды
Запуск локальной Ethereum-ноды посредством инструмента **Anvil (Foundry)**. Не требует синхронизации с сетью, пригоден для воспроизводимого тестирования.

### 2. Профилирование системных вызовов
Анализ syscall-профиля процесса ноды с использованием `strace` и `perf`. Составление перечня системных вызовов, необходимых для штатной работы.

### 3. Разработка seccomp-профиля
Составление whitelist-фильтра системных вызовов в формате **seccomp BPF**. Все вызовы, не входящие в перечень, блокируются на уровне ядра.

### 4. Применение механизмов изоляции
Запуск ноды с применением:
- **seccomp**-фильтра
- **network namespace** (ограничение сетевого взаимодействия)
- **cgroups** (лимитирование потребления CPU и RAM)

### 5. Демонстрация модели угроз
Сравнительная демонстрация поведения системы при симулируемой атаке — без изоляции и с применёнными механизмами защиты. Фиксация заблокированных действий.

### 6. Документирование результатов
Оформление технического отчёта и репозитория на GitHub: описание архитектуры решения, конфигурационные файлы, скрипты, выводы.
